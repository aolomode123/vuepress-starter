# 智慧城市操作手册

# 1. 设备管理DV

一个设备需要正常运行，需要用户按照以下流程，将设备导入到资产，然后在设备中进行添加，最后进行业务回路配置才能在业务环境中运行。

所有设备进入实际业务运营环境中运行，需要按照如下流程进行设备的录入。

![[资产设备导入流程.png]]

操作步骤：

  1. 在“资产”中添加资产。
  2. 在“设备”中添加设备，绑定设备之间的业务关联关系。
  3. 在回路控制箱设备详情中，配置回路参数，包括单灯回路归属。

在物资模块中，导入物资，你可以通过页面导入单个或者批量导入物资。

  4. 节能率：相比于节能前钠灯的耗能所计算得出的节能率。

![[回路控制箱详情-控制.png]]

  1. 设备控制、调试
  • 全开、全关：回路控制箱下的所有回路开关操作。
  • 召测：对设备进行远程状态查询，查看设备是否离线、故障。
  • 设置状态采集：对设备的数据进行采集，查询设备的相关运行的数据情况。
  1. 回路开关：对回路控制箱下的单个回路进行开关操作。

# 回路归属

回路归属的目的是对回路下的单灯进行后期归属，因为前期添加设备的时候，用户手动添加到回路时并不一定正确，而且随着后期的运维后，单灯的维修、更换、迁移等都会导致归属发生变化，因此回路归属可以帮助进行归属更新，利于路灯的运维。

“回路归属时长”可根据回路下的单灯数量自定义填写，默认1分钟，建议100个单灯5分钟以上。

![[添加回路控制箱回路归属.png]]

# 2. 分组管理

分组有两种类型的设备分组，即“回路分组”、“单灯分组”。

分组的目的是将分组的设备进行策略配置，达到整体组控的目的，例如可以对全程的回路进行每日“定时开关灯”，或者对全程的单灯进行“半夜调光50%”的节能策略，从而达到对城市道路照明设备进行智能控制和节能的目的。

# 3. 任务策略

在城市道路照明系统中，存在两种最关键的设备，即回路控制箱和单灯控制器，需要对这些设备分配相关的策略进行操控，达到统一开关灯或者节能的目的。

# 3.1 回路控制器策略

城市路灯设备的每日开关灯是通过对回路控制箱的整体策略配置实现的，控制方式有多种。

定时：通过设定每日的开关灯时间进行控制。 晨昏：通过当前位置经纬度的获取晨昏时间进行开关控制。 手动：如果手动模式下，设备无任何智能策略，需要用户手动开关灯。 光照：需要联动光照仪进行开关灯，可设定阈值来控制策略。另外，光照采集仪还可以和定时开关进行组合，例如：开灯用“光照”，关灯用“定时”。

回路控制箱策略配置流程如下。

![[策略流程.png]]

以下为回路控制箱的策略配置步骤。

# （1）对回路进行分组

在进行回路策略配置之前，到左侧菜单“设备管理/分组管理/回路分组”中对需要回路进行分组。

![[分组管理-回路分组.png]]

# （2）回路控制箱策略配置

回路控制箱的策略配置参数步骤，如下操作。

![[新增规则.png]]

按照如下步骤配置参数。

![[新增规则-回路-定时-参数.png]]

  1. 名称：填写个通俗易懂的策略名称，例如“少林路回路每日开关灯策略”。
  2. 应用设备对象：选“回路控制箱”。
  3. 分组：选择策略应用到的分组，如果没有分组，请到设备管理/分组中对回路进行分组。
  4. 生效日期：策略在此日期范围内才会生效。
  5. 可选择定时/晨昏/手动/光照，用户可根据自己的需求进行选择。
  6. 多时段定时开关：可选单时段/多时段，用户可根据自己需求选择。
  7. 下发配置：将上述配置的参数下发到回路中。
  8. 批量下发：经过上述配置测配置下发后已经完成配置操作，如果后期需要部分对某些回路进行批量配置，那么可通过“批量下发”再次进行部分配置下发，下图所示。

# 5. 告警配置

告警配置流程如下3个步骤所示。

![[告警配置流程.png]]

告警配置规则功能页面如下所示。

![[创建告警规则-大纲步骤.png]]

# 5.1 基本设置

下图为“基本设置”页面。

![[创建告警规则-1基本设置.png]]

  1. 告警名称：给告警规则取个通俗易懂名称，方便后期管理，比如“单灯控制器高压”。
  2. 告警对象：选择需要配置告警规则的设备对象，设备对象包含回路控制箱、回路、单灯控制器，请根据自己需求选择。

# 5.2 告警规则

用户可根据实际业务情况进行告警规则配置，比如现场电压满足用户配置的 “大于280V时触发告警”，那么触发高压告警通知，用户可以选择告警的通知方式有“系统、小程序、短信”。

下图为“告警规则”页面。

![[创建告警规则-2告警规则.png]]

  1. 告警判断规则：根据实际业务的场景需求设置告警触发判断规则，比如电压，实际业务下要求“超过280V”就属于高压，需要出发报警。
  2. 告警条件类型：根据实际业务场景进行配置即可，比如某些场景下，如果电压持续高压超过10分钟，那么才能触发告警。请用户根据自己需求设置，条件包括 “直接上报/累计上报次数/持续上报时间”。

# 5.3 消警规则

消警规则的目的是自动对已经触发的告警信息进行消警，当设备正常恢复到消警条件下后，会自动消除告警，不会在展示在前段告警中，但会保留在日志中供查询需要。注：消警规则非必填，可根据实际情况进行配置。

下图为“消警规则”页面。

![[创建告警规则-3消警规则.png]]

  1. 告警判断规则：根据实际业务的场景需求设置告警触发判断规则，比如电压，实际业务下要求“超过280V”就属于高压，需要出发报警。
  2. 告警条件类型：根据实际业务场景进行配置即可，比如某些场景下，如果电压持续高压超过10分钟，那么才能触发告警。请用户根据自己需求设置，条件包括 “直接上报/累计上报次数/持续上报时间”。

# 6. 告警消息

# 6.1 电脑端

![[首页-告警消息列表.png]]

  4. 批量操作：如果需要批量进行消息处理和推送，可用批量操作来快速处理。
  5. 批量推送：该功能仅供管理员使用，用来推送重要告警消息给到运维人员的小程序。

# 8. 数据大屏
